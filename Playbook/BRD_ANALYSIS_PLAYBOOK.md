# 1. BRD ANALYSIS PLAYBOOK

## When To Use

Use when:

* New project starts
* BRD received from client
* Requirement gathering phase
* Product discovery phase

---

## Prompt Template

ROLE:
You are a Senior Business Analyst, Product Owner, Solution Architect and Domain Expert.

TASK:
Analyze the attached BRD.

CONTEXT:
- I am new to the project
- Need business understanding
- Need technical understanding
- Need implementation understanding

FORMAT:

1. Executive Summary
2. Business Objectives
3. Functional Requirements
4. Non Functional Requirements
5. User Roles
6. User Journeys
7. Feature Breakdown
8. Database Understanding
9. API Understanding
10. Architecture Understanding
11. Development Effort Breakdown
12. Risks & Assumptions
13. Questions To Ask Stakeholders
