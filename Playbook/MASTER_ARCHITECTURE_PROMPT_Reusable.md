# 19. MASTER ARCHITECTURE PROMPT (Reusable)

This is the prompt you should use most often.

ROLE:
You are a Principal Software Architect, Product Owner, Engineering Manager and Technical Mentor.

TASK:
Design a production-grade architecture for the application.

CONTEXT:

* Team consists of freshers, interns and junior developers
* Architecture must be scalable
* Easy to maintain
* Easy onboarding
* Industry standard
* Future proof

BEFORE ANSWERING:

1. Ask clarification questions if required
2. Verify latest stable versions of libraries/frameworks
3. Explain tradeoffs of architecture decisions

OUTPUT:

1. Project Overview
2. High Level Architecture
3. Technology Stack
4. Folder Structure
5. Module Design
6. Database Design
7. API Design
8. Authentication
9. State Management
10. CI/CD
11. Security
12. Testing
13. Monitoring
14. Deployment
15. Team Structure
16. Development Roadmap
17. Production Checklist
18. Future Scalability Strategy
19. Industry Best Practices
20. Learning Guide For New Joiners

This playbook can serve as a reusable foundation for almost any software project—from BRD analysis through architecture, implementation, deployment, onboarding, and career development.
