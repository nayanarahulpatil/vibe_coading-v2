Persona understanding

- You can create multiple Persona md files and AI can load the appropriate persona for the task.
- Different experts thinks differently hence Persona matters.
- Verious Persona patterns
1. Expert Persona
    Best for Coding, Architecture, Debugging.
    Example:- Act as a Senior React Native Architect.
2. Reviewer Persona
    Best for Code review, Architecture review.
    Example:- Act as a Principal Engineer reviewing code.
3. Teacher Persona
    Best for Learning, Interview preparation, Understanding concepts
    Example:- Act as a React Native mentor.
4. Interviewer Persona
    Best for Mock interviews, DSA practice
    Example:- Act as FAANG interviewer.
5. Cognitive Verifier Persona
    Best for Production systems, Critical architecture decisions
    Example:- Act as a senior engineer and verify every assumption before giving the answer.
- Multi-Persona workflow example
Step 1
    Role:- Product Owner
    Create BRD for Food Delivery App
    Output:- BRD
Step 2
    Role:- Software Architect
    Analyze BRD and create architecture.
    Output:- Architecture
Step 3
    Role:- Backend Engineer
    Create APIs from architecture.
    Output:- API Specifications
Step 4
    Role:- React Native Developer
    Create mobile implementation plan.
    Output:- Frontend Plan
Step 5
    Role:- QA Engineer
    Generate test cases.
    Output:- Testing Plan
